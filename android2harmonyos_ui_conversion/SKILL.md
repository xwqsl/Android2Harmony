# Android to HarmonyOS UI Conversion Skill

## Overview
This skill provides comprehensive knowledge and guidelines for converting Android UI components and patterns to HarmonyOS (OpenHarmony) UI system. It covers all major UI dimensions including component mapping, layout structures, routing, lifecycle management, and event handling.

## Functionality
- **Component Mapping**: Detailed mapping between Android UI components and HarmonyOS equivalents
- **Layout Systems**: Guidelines for converting Android layouts to HarmonyOS layouts
- **Page Routing**: Techniques for implementing Android Intent-based navigation in HarmonyOS
- **Lifecycle Management**: Mapping Android activity/fragment lifecycles to HarmonyOS abilities/pages
- **Event Handling**: Converting Android click events and gesture handling to HarmonyOS equivalents
- **Resource Management**: Strategies for converting Android resource files to HarmonyOS format
- **Theme and Styling**: Guidelines for implementing Android themes in HarmonyOS

## Usage
This skill is intended to help AI models understand the differences between Android and HarmonyOS UI systems and provide accurate conversion recommendations. It can be used as a reference when converting Android apps to HarmonyOS or when answering questions about the conversion process.
